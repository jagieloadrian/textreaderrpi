<objective>
Consolidate routing into feature-first modules (text, display, health, metrics placeholder) and verify rate-limit coverage on all API routes including text submission.
</objective>

<context>
Current routing assembly in `Routing.kt` delegates to `textRoutes()`, `displayApiRoutes()` (from web/routes/), and `webRoutes()`. The rate limiter is applied at the `/api` route level, which already covers text and display routes. This plan consolidates display API routes into the main routing package, creates a dedicated health routes module, and verifies/documents rate-limit coverage.
</context>
