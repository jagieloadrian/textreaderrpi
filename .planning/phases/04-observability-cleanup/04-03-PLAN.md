<objective>
Align health stack by keeping KHealth as the core health check engine (D4-07), ensuring `/health` and `/health/ready` semantics remain stable (D4-09), and extending health responses with Phase 3 fields (uptime, memoryUsedMb, memoryMaxMb, displayType, isActive, lastError) via an adapter/bridge pattern (D4-20).
</objective>

<context>
KHealth is already installed in `Monitoring.kt` with `healthChecks { check("appAlive") { true } }` and `readyChecks { check("displayReady") { screenDriverService.status().hardwareAvailable } }`. The `/health/detail` route was created in 04-01 as a separate endpoint. This plan enriches it with the full D4-08 contract and ensures KHealth remains the authoritative health core. D4-24 locks the explicit KHealth checks as baseline.
</context>

<task id="2" name="Implement full health detail logic in HealthRoutes">
<read_first>
- src/main/kotlin/com/anjo/routing/HealthRoutes.kt
- src/main/kotlin/com/anjo/service/ScreenDriver.kt
- src/main/kotlin/com/anjo/di/Monitoring.kt
</read_first>
<action>
Update `HealthRoutes.kt` `GET /health/detail` handler to compute:
- `status`: "UP" if `screenDriverService.status().hardwareAvailable` is true, else "DOWN"
- `uptime`: `ManagementFactory.getRuntimeMXBean().uptime` (Long, milliseconds)
- `memoryUsedMb`: `(Runtime.getRuntime().totalMemory() - Runtime.getRuntime().freeMemory()) / (1024 * 1024)`
- `memoryMaxMb`: `Runtime.getRuntime().maxMemory() / (1024 * 1024)`
- `displayType`: `screenDriverService.currentDisplayType()`
- `isActive`: `screenDriverService.status().hardwareAvailable`
- `lastError`: `screenDriverService.status().error`

Import `java.lang.management.ManagementFactory`. Respond with `HealthDetailResponse`.
</action>
<acceptance_criteria>
- `GET /health/detail` returns JSON with all 7 fields populated
- `uptime` is a positive Long (milliseconds)
- `memoryUsedMb` and `memoryMaxMb` are positive Longs
- `displayType` matches the configured display type (e.g., "MAX7219")
- `lastError` is null when no errors have occurred
- `./gradlew compileKotlin` passes
</acceptance_criteria>
</task>
