# Plan 05-06: Schedule HTTP CRUD API Routes + Input Validation

## Objective

Implement the Schedule HTTP CRUD API (D5-11):
- `POST /api/schedule` — create a new schedule with validation
- `GET /api/schedule` — list all schedules
- `GET /api/schedule/{id}` — get a specific schedule by id
- `DELETE /api/schedule/{id}` — delete and cancel a schedule
- `PATCH /api/schedule/{id}` — update a schedule (enable/disable/change text)

Input validation per security constraints:
- Validate `triggerType` is ONESHOT, RECURRING, or CRON
- Validate `triggerValue` format matches triggerType (ISO8601 date, interval string, or cron expression)
- Validate `effect` enum value; reject invalid with 422
- Validate `priority` range: 0..100
- Validate `text` length ≤ 512 characters
- Reject invalid cron expressions using cron-utils parse (return 422 with message)

## Background

This plan adds the REST API surface for schedule management. Route handlers use the `closestDI()` pattern to inject `ScheduleRepository` and `SchedulerService`. Input validation happens in the route handler before DB write.

    Create `src/main/kotlin/com/anjo/routing/ScheduleRoutes.kt`.
    Package: `package com.anjo.routing`
    Imports: Ktor routing/http/application, model, db, service imports; `com.cronutils.parser.CronParser`, `com.cronutils.model.definition.CronDefinitionBuilder`, `com.cronutils.model.CronType`

    Define `fun Application.configureScheduleRoutes()` (mirror the pattern from TextRoutes.kt exactly).

    Inside, inject via DI:
    - `val repository by closestDI().instance<ScheduleRepository>()`
    - `val schedulerService by closestDI().instance<SchedulerService>()`

    Add a `private val cronParser = CronParser(CronDefinitionBuilder.instanceDefinitionFor(CronType.UNIX))`

    Register these routes inside `routing { }`:

    `GET /api/schedule`:
    - `val schedules = repository.findAll()`
    - `call.respond(schedules)`

    `POST /api/schedule`:
    - `val body = call.receive<Schedule>()`
    - Validate `body.text.length <= 512` — else `return@post call.respond(HttpStatusCode.UnprocessableEntity, "text too long")`
    - Validate `body.priority in 0..100` — else 422 "priority out of range 0..100"
    - Validate `body.effect` is valid Effect enum — else 422 "invalid effect"
    - Validate `body.triggerType` is valid TriggerType — else 422
    - If `body.triggerType == TriggerType.CRON`: try `cronParser.parse(body.triggerValue)` — catch exception → 422 "invalid cron expression: ${e.message}"
    - If `body.triggerType == TriggerType.RECURRING`: validate value matches pattern `^\d+[smhd]$` — else 422 "invalid interval format; use e.g. 5m, 2h, 30s"
    - If `body.triggerType == TriggerType.ONESHOT`: try `Instant.parse(body.triggerValue)` — catch → 422 "invalid datetime; use ISO8601"
    - Call `repository.insert(body)` → `val created = ...`
    - Call `schedulerService.schedule(created)` to start the coroutine job
    - `call.respond(HttpStatusCode.Created, created)`

    `GET /api/schedule/{id}`:
    - `val id = call.parameters["id"] ?: return@get call.respond(HttpStatusCode.BadRequest, "missing id")`
    - `val schedule = repository.findById(id) ?: return@get call.respond(HttpStatusCode.NotFound)`
    - `call.respond(schedule)`

    `DELETE /api/schedule/{id}`:
    - `val id = call.parameters["id"] ?: return@delete call.respond(HttpStatusCode.BadRequest, "missing id")`
    - `schedulerService.cancel(id)` — cancels the running coroutine job
    - `val deleted = repository.delete(id)`
    - `if (!deleted) return@delete call.respond(HttpStatusCode.NotFound)`
    - `call.respond(HttpStatusCode.NoContent)`

    `PATCH /api/schedule/{id}`:
    - `val id = call.parameters["id"] ?: return@patch call.respond(HttpStatusCode.BadRequest, "missing id")`
    - `val body = call.receive<Schedule>()`
    - `val updated = repository.update(id, body) ?: return@patch call.respond(HttpStatusCode.NotFound)`
    - If schedule is active and text/trigger changed: cancel and re-schedule: `schedulerService.cancel(id); schedulerService.schedule(updated)`
    - `call.respond(updated)`
  </action>
