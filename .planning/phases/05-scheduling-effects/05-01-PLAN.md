# Plan 05-01: Health Endpoint Consolidation + Retry Replacement

## Objective

Merge `/health/detail` into `/health` via KHealth extension mechanism (D5-01).
Replace `RecoveryPolicy` class with an in-house `retryWithBackoff` suspend function (D5-02).
Move retry configuration to `application.yaml` under `retry.*` namespace (D5-03).

## Background

Phase 4 added a separate `/health/detail` endpoint returning extended fields (uptime, memoryUsedMb, memoryMaxMb, displayType, isActive, lastError). Decision D5-01 consolidates these into the primary `/health` response. The custom `RecoveryPolicy` class is replaced by a lightweight coroutine-native retry function configured from YAML (D5-02, D5-03).

<task id="05-01-02" type="execute">
  <title>Create retryWithBackoff suspend function and RetryConfig data class</title>

  <read_first>
    - src/main/kotlin/com/anjo/service/RecoveryPolicy.kt — read full current implementation to understand what to replace
    - src/main/resources/application.yaml — read current config structure to know where to add retry.* section

  </read_first>

  <action>
    Create a new file `src/main/kotlin/com/anjo/service/RetryPolicy.kt`.
    Package declaration: `package com.anjo.service`
    Imports needed: `kotlinx.coroutines.delay`, `org.slf4j.LoggerFactory`, `kotlin.math.min`

    Define a data class `RetryConfig` with fields:
    - `maxAttempts: Int` (default 5)
    - `initialDelayMs: Long` (default 1000)
    - `maxDelayMs: Long` (default 30000)
    - `factor: Double` (default 2.0)

    Define a top-level suspend function `retryWithBackoff` with signature:
    `suspend fun <T> retryWithBackoff(config: RetryConfig = RetryConfig(), block: suspend () -> T): T`

    The function logic:
    - Track `attempt` counter starting at 0, `delayMs` starting at `config.initialDelayMs`
    - Loop: try executing `block()`, return result on success
    - On exception: increment attempt; if attempt >= config.maxAttempts, rethrow
    - Otherwise: log warning with attempt number and delay, call `delay(delayMs)`, then update delayMs = min(delayMs * config.factor, config.maxDelayMs).toLong()
    - Logger: `private val log = LoggerFactory.getLogger("com.anjo.service.RetryPolicy")`

    Do NOT delete RecoveryPolicy.kt yet (task 05-01-04 will handle that after call sites are migrated).
  </action>

  <acceptance_criteria>
    - `src/main/kotlin/com/anjo/service/RetryPolicy.kt` exists
    - File contains `data class RetryConfig(`
    - File contains `suspend fun <T> retryWithBackoff(`
    - File contains `delay(delayMs)`
    - `./gradlew compileKotlin` exits 0 (compiles without errors)
  </acceptance_criteria>
</task>
