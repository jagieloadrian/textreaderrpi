# Plan 05-08: Effect Renderer Architecture + DisplayDriver Interface Extensions

## Objective

Design the `EffectRenderer` strategy interface and four implementations: `ScrollEffect`, `BlinkEffect`, `ReverseEffect`, `FadeEffect` (D5-19, D5-21).
Extend the `DisplayDriver` interface with `setBrightness(level: Int)` and `displayStatic(text: String)` operations required by blink and fade effects.
Implement these new operations in all concrete `DisplayDriver` implementations (MAX7219, LCD, OLED, Offline/Simulated).
Create `EffectRendererFactory` to map `Effect` enum to implementation.
Wire `EffectRendererFactory` into DI.

## Background

Effects decorate the rendering pipeline via the strategy pattern. The `EffectRenderer` interface has one method: `suspend fun render(text: String, driver: DisplayDriver)`. Each implementation applies the effect using the driver's API. The `ScreenDriverService` selects the correct renderer via `EffectRendererFactory` before triggering display. The MAX7219 supports hardware brightness via register 0x0A (intensity 0–15); blink and fade use this register.
