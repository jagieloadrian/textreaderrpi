# Plan 05-09: Wire Effect Field into POST /api/text and POST /api/schedule

## Objective

Add the optional `effect` field to `POST /api/text` request body (D5-20).
When `effect` is provided, the `ScreenDriverService` selects the correct `EffectRenderer` from `EffectRendererFactory`.
Validate that the provided effect value is a known `Effect` enum member; reject with 422 otherwise.
`POST /api/schedule` already sends the effect field through `ScheduleRoutes.kt` (plan 05-06); verify it is used by `SchedulerService.fire()` via `EffectRendererFactory`.

## Background

Before this plan, `POST /api/text` always scrolled text with no effect. After this plan, callers can pass `{"text": "hello", "effect": "FADE"}` to trigger a fade-in or `{"text": "hello", "effect": "BLINK"}` for blink. The default when effect is absent remains `SCROLL`.
