# Plan 05-02: Mutex Concurrency Guard + Config-Driven Metrics + Comment Removal

## Objective

Replace `ResourceTracker` with `kotlinx.coroutines.sync.Mutex` for display concurrency protection in `ScreenDriverService` and remove the class entirely (D5-04).
Make `MetricRegistry` instrumentation config-driven from `application.yaml` under `metrics.*` namespace with an enabled flag and a name prefix (D5-05).
Remove all comments (block and line) from every service and driver source file (D5-06).

## Background

`ResourceTracker` was built as a slot-counting concurrency guard, but `ScreenDriverService` already uses an `AtomicBoolean` for busy-tracking. D5-04 replaces both with a single `Mutex` — the idiomatic coroutine concurrency primitive. `ResourceTracker` is confirmed removable.

`ScreenDriverService` currently hardcodes metric names and registers them directly in its constructor body. D5-05 introduces a `MetricsConfig` (enabled + prefix) loaded from YAML and threaded through via constructor so that metric names use the configured prefix and the entire instrumentation can be disabled without touching service logic.

D5-06 strips every `//` line comment and every `/** */` / `/* */` block comment from all files under `src/main/kotlin/com/anjo/service/` and `src/main/kotlin/com/anjo/driver/`. Code clarity replaces comment noise.
