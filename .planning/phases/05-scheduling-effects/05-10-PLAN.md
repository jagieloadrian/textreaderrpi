# Plan 05-10: Timing-Accurate Behavior Tests

## Objective

Write timing-accurate behavioral tests for:
1. **Scheduler firing** — one-shot fires at target time, recurring fires at interval, cron fires at next cron time (D5-22)
2. **Conflict policy** — ad-hoc preempts scheduled display; interrupted schedule is re-queued; priority tie-break works (D5-22)
3. **Effect rendering** — Scroll, Blink, Reverse, Fade each call the correct DisplayDriver methods in the correct sequence (D5-22)
4. **Retry behavior** — retryWithBackoff retries the correct number of times with backoff delays

All scheduler and retry tests use virtual time via `kotlinx-coroutines-test` `TestCoroutineScheduler.advanceTimeBy()` — NO real `Thread.sleep()` or wall-clock waits.

## Background

Decision D5-22 explicitly requires "timing-accurate behavior tests." The standard approach in Kotlin is `runTest { testScheduler.advanceTimeBy(millis) }` which advances the virtual clock without real delays — tests run in milliseconds even for "60-second" scheduler intervals. This plan creates all tests that verify the core behavioral contracts of the scheduling + effects system.
