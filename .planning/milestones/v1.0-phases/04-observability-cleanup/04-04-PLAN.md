<objective>
Implement `GET /metrics` JSON endpoint returning runtime, API, and hardware metric groups in a dynamic grouped schema. The endpoint is rate-limited with a dedicated milder policy and designed for future Micrometer migration.
</objective>

<context>
The project already uses DropwizardMetrics `MetricRegistry` (installed in Monitoring.kt, injected via DI). ScreenDriverService already registers meters/counters/timers with the registry. The metrics endpoint serializes this existing data into a structured JSON format. Rate limiting for `/metrics` uses the dedicated helper created in 04-01.
</context>

    val groups: List<MetricGroup>

@Serializable data class MetricGroup(

    val count: Long? = null,

)
