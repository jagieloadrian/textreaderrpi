## Tasks

### T11-01: SchedulerService bug fix — cancel-on-schedule logic
**File:** `src/main/kotlin/com/anjo/service/SchedulerService.kt`

Root cause: `schedule()` called `cancel(id)` which unconditionally wrote `DONE` to the DB for every schedule — including brand-new ones. `tickLoop` filters by ACTIVE status, so newly created schedules were immediately invisible to the tick loop.

Fix: `schedule()` now only removes/cancels the in-memory coroutine (`activeJobs.remove(id)?.cancel()`) without touching DB status. `cancel()` is reserved exclusively for external/manual cancellation (API route) and is the only place that writes `DONE` to DB.

### T11-02: New cancel endpoint `POST /api/v1/schedule/{id}/cancel`
**File:** `src/main/kotlin/com/anjo/routing/ScheduleRoutes.kt`

Added `post("/cancel")` sub-route under `/{id}`:
- Calls `schedulerService.cancel(id)` → stops coroutine + writes DONE to DB
- Does NOT delete the record — schedule stays in history with status DONE
- Returns `204 No Content` (idempotent for unknown ids)

### T11-03: UI — Stop button for active recurring/cron schedules
**File:** `src/main/resources/static/app.js`

Added `stopSchedule(id)` function calling `POST /api/v1/schedule/{id}/cancel`.
Schedule table rows for `status === ACTIVE && (triggerType === RECURRING || triggerType === CRON)` show a **Stop** action link alongside the existing **Delete** action.
