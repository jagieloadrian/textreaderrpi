# Plan 05-07: Schedule UI Page + DI Wiring + Lifecycle Verification

## Objective

Implement the schedule management UI page at `GET /schedule` using Ktor HTML DSL templates (D5-14).
The page lists active schedules and provides a form to create new ones.
Verify that the complete DI and lifecycle wiring from plans 05-04 through 05-06 works end-to-end.

## Background

The schedule UI page is a server-rendered HTML page using the same Ktor HTML DSL template system established in Phase 2. It follows the same `respondHtml { }` pattern as the `IndexPage`, `StatusPage`, and `SettingsPage` templates — a consistent base layout with route-specific page content.

    5. The form's action is `POST /api/schedule` (JSON POST — so either use JavaScript `fetch` or accept form-encoded input and convert)
